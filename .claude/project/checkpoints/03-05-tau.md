## Toolkit deltas since previous checkpoint

Captured during tau capstone authoring; full list logged in `03-core-tau-diagrams.md` under [Toolkit refinements applied](../03-core-tau-diagrams.md#toolkit-refinements-applied) → "tau capstone (2026-05-11)".

`diagram-ingredients` skill:

- **Mirrored L-shapes with same first-segment direction** (`references/edges-and-marks.md`) — Fletcher's auto `label-side` for a polyline resolves against segment 0's tangent θ. Two L-shapes that *start* in the same direction (e.g., both go south, then one turns east and the other turns west) auto-resolve to the same `label-side`, which is correct for one and inverted for its mirror. Override the inverted case with explicit `label-side: left` (or `right`). Extends the existing "Mirrored label placement" guidance with the polyline-specific failure mode. The empirical rule traced through Fletcher's source: `θ-normal = θ + 90deg` for `label-side: right`, `θ - 90deg` for `label-side: left`; the label position is `curve-point + (-label-sep, θ-normal)`. For the first segment of a south-then-east vs south-then-west pair, both segments have θ = -90° (south), so both auto-resolve identically.

- **Center-overlay when both sides are occupied** (`references/edges-and-marks.md`) — when an edge sits between two flanking labeled edges, use `label-side: center` to overlay the label on the line; `label-fill` masks the line beneath. The pattern was driven by the `cfg (passed through)` edge in registry-composition, which is flanked by the two `Create()` diagonals on its left and right.

`typst-diagrams` skill:

- **Wide in-diagram nodes inflate the column they sit in** (`references/fletcher-pitfalls.md`) — Fletcher's grid computes column width as the max node width at that column across every row, so a wide caption/legend node at col 0 widens col 0 for all rows above it too. Workaround: move supplementary content (legends, footnotes) outside the `#diagram(...)` call as page-level Typst content (centered with `#align(center, ...)`); the auto-sized page width accommodates it without disturbing the grid. Fallback: place the node at a fractional coord between existing columns (e.g., `(2.5, n)`) which creates a new column that other rows don't populate. This pitfall surfaced in the provider-format-matrix diagram during the first legend attempt; the page-level form became the convention.
